# Changes — 2026-06-09 (algorithm & backtest fidelity fixes)

Implements Tier 1/2 items from `ANALYSIS_ALGO_IMPROVEMENTS.md`. All changes verified by syntax checks, the existing strategy test suite (18 passed), and a synthetic-bar smoke test of the new exit logic.

## backtest.py — engine fidelity

**Intrabar exit triggers (default ON — re-baseline required).** `check_exits()` now triggers stops on the bar's adverse extreme (low for longs) and targets on the favorable extreme, matching the live monitor which evaluates live prices. Previously a 4h bar whose low pierced the stop but closed above it never stopped out — systematically inflating WR/P&L. When both stop and target are touched in one bar, the stop wins (conservative). Legacy behavior: `--no-intrabar-exits`. **All pre-existing backtest results and leaderboard rankings are optimistic and not comparable to new runs.**

**Slippage modeling.** `--slippage-pct` (default 0.0) applies adverse slippage to every fill in `_compute_pnl()`. 0.1–0.3 is realistic for the mid-cap pairs this bot trades.

**Out-of-sample windows.** `--end-days-ago N` ends the data window N days before now. Enables train/validate splits, e.g. train `--days 1460 --end-days-ago 365`, validate `--days 365`. Cache still downloads to now; only trimming changes.

**Breakeven false branch implemented.** With `--no-breakeven-requires-tp1`, breakeven now arms early at `--breakeven-trigger-r` (default 0.5R), mirroring monitor.py's guard path. Previously the false branch didn't exist, making the parameter untestable (DIAGNOSIS_PARAMETER_PASSTHROUGH.md).

**HTF RSI gate de-tautologized.** `htf_rsi_bars_interval` default changed 4h → 1h (live parity with `strategy.py`'s `htf_interval`). A WARNING prints if it equals the entry interval (the configuration that made exp_004/exp_005 no-ops).

**Swing stop option.** `--swing-stop-recent` anchors the swing stop to the most recent swing low instead of the lowest swing in the whole window (both in `_compute_stop_and_targets()` and the htf_trend entry path). Default off (legacy). High-value experiment — changes stop distance and the entire R framework.

**New CLI flags.** `--tp1-R`, `--tp2-R`, `--max-bars-in-trade`, `--reversal-body-pct`, `--breakeven-trigger-r`, `--dev-threshold-pct` (alias of `--dev-threshold`) — all previously requested in HUMAN_NOTES "CLI Flags Needed" / "Skipped Proposals". Generic exit knobs merge for ALL strategies; entry-filter knobs remain vwap-only.

## Strategy code (research/strategies/)

**meanrev: ADX regime gate added to `generate_signals()` — LIVE BEHAVIOR CHANGE.** The strategy's documented core premise ("mean reversion FAILS in trending markets; the ADX filter is essential") was enforced in `evaluate()` (screener scoring) and in backtest.py's entry check, but NOT in the path that fires live trades. Now blocks entries when ADX ≥ `adx_max_threshold` (same semantics as backtest: passes if ADX not computable). Bar buffer enlarged to 60 for stable ADX. This aligns live behavior with what backtests validate.

**vwap_meanrev: screener/runner criteria aligned.** `evaluate()` now uses the same deviation mode as `generate_signals()` (percentage-based per Ross Cameron spec when `use_percentage_deviation=True`). Previously confidence scores were computed from ATR-based deviation while trades fired on percentage deviation.

**vwap_meanrev: regime filter can actually block (default OFF).** New config `regime_block_bearish` — when true, blocks longs if HTF price < EMA200 AND EMA200 slope strongly bearish. The legacy "1h regime filter" computed the trend then returned allowed=True on every path; only the volatility cap ever blocked. Default false preserves current live behavior; enable via config/experiment after backtesting.

**vwap_meanrev + htf_trend: `swing_stop_recent` config (default OFF).** Mirrors the backtest flag for live parity when enabled.
