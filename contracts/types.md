# Contract Types (Authoritative)

This document defines the shared domain types exchanged between system components.
These types are **authoritative** and must be referenced by Backend, Frontend, and Quant code.

No component may redefine these structures independently.

---

## TradeIntent

**Purpose:**  
Represents a strategy's request to enter or exit a position.

**Produced by:** Strategy Modules  
**Consumed by:** Risk Manager

**Fields (v1):**

| Field | Type | Description |
|-------|------|-------------|
| `strategy_id` | `string` | Unique identifier for the strategy that generated this intent. Must match a registered strategy in the `strategies` table. |
| `symbol` | `string` | Trading pair symbol (e.g., "BTC/USD", "ETH/USD"). Must match Kraken symbol format. |
| `side` | `enum: "buy" \| "sell"` | Direction of the trade intent. "buy" indicates entering a long position or closing a short. "sell" indicates entering a short position or closing a long. |
| `intent_type` | `enum: "enter" \| "exit" \| "reduce"` | Type of intent. "enter" opens a new position. "exit" closes an entire position. "reduce" partially closes a position. |
| `notional_risk_pct` | `number` | Percentage of total account equity to risk on this trade. Must be a positive number. Represents the maximum loss exposure if the trade moves against the position. |
| `metadata` | `object` | Opaque key/value map for strategy-specific context. May include indicators, signal strength, or other strategy-internal state. No schema enforcement; strategies are free to include any JSON-serializable data. |

**Notes:**
- TradeIntents express *desire*, not permission.
- No execution-specific fields allowed (price, order type, nonce).
- The Risk Manager evaluates `notional_risk_pct` against current portfolio exposure and strategy limits.

## RiskDecision

**Purpose:**  
Represents the Risk Manager's evaluation of a TradeIntent.

**Produced by:** Risk Manager  
**Consumed by:** Execution Engine

**Fields (v1):**

| Field | Type | Description |
|-------|------|-------------|
| `intent_id` | `string` | Unique identifier linking this decision to the original TradeIntent. Used for traceability and audit. |
| `approved` | `boolean` | Whether the TradeIntent is approved for execution. `true` means the intent passes all risk checks and may proceed. `false` means the intent is rejected. |
| `rejection_reason` | `string \| null` | If `approved` is `false`, this field contains the reason for rejection. Common values: "exceeds_portfolio_limit", "exceeds_strategy_limit", "system_halted", "stale_market_data". If `approved` is `true`, this field must be `null`. |
| `evaluated_portfolio_risk` | `number` | The calculated portfolio exposure percentage at the time of evaluation. Represents `(total_exposure / total_equity) * 100`. Used for audit and debugging. |
| `timestamp` | `string` (ISO8601) | UTC timestamp when the risk evaluation was performed. Format: `YYYY-MM-DDTHH:mm:ssZ` (e.g., "2024-01-01T12:00:00Z"). |

**Notes:**
- Risk Manager defaults to fail-closed: if evaluation cannot be completed (e.g., missing data), `approved` must be `false`.
- `evaluated_portfolio_risk` reflects the state *after* this intent would be applied (includes pending intents).

## Fill

**Purpose:**  
Represents the result of an executed order.

**Produced by:** Execution Engine  
**Consumed by:** Persistence, Frontend

**Fields (v1):**

| Field | Type | Description |
|-------|------|-------------|
| `order_id` | `string` | Internal order identifier generated by the Execution Engine. Unique within the system. Used for tracking and audit. |
| `symbol` | `string` | Trading pair symbol that was executed (e.g., "BTC/USD", "ETH/USD"). Must match the TradeIntent symbol. |
| `side` | `string` | Execution side: "buy" or "sell". Must match the TradeIntent side. |
| `executed_price` | `number` | Average price at which the order was filled. For partial fills, this is the volume-weighted average price. |
| `quantity` | `number` | Quantity of the base asset that was executed. Must be positive. |
| `fees` | `number` | Total fees paid for this execution, denominated in the quote currency (e.g., USD). Must be non-negative. |
| `slippage` | `number` | Slippage amount, calculated as the difference between intended price and executed price. Positive values indicate worse execution (buy higher, sell lower). Denominated in quote currency. |
| `exchange_order_id` | `string` | Order identifier returned by the exchange (Kraken). Used for reconciliation and support. Must be unique per exchange. |
| `timestamp` | `string` (ISO8601) | UTC timestamp when the order was executed on the exchange. Format: `YYYY-MM-DDTHH:mm:ssZ` (e.g., "2024-01-01T12:00:00Z"). |

**Notes:**
- Fill objects are immutable once created.
- Partial fills may result in multiple Fill objects for a single order_id.
- All monetary values (executed_price, fees, slippage) are in the quote currency.
