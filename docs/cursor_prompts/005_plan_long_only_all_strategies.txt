## The Established Pattern (from vwap_meanrev fix)

1. Add `long_only: bool = True` to the strategy's Config dataclass
2. Add `"long_only": True` to DEFAULT_CONFIG in backtest.py if a shared config
   is used (or to the strategy-specific config dict if separate)
3. Gate the short branch in the backtest entry function via cfg.get("long_only", True)
4. Ensure the live strategy class respects the config flag too (check that
   strategy.py's signal generation has equivalent gating)
5. Tests: confirm no test breaks; if a test asserts shorts are produced in
   pipeline mode, flag it (don't fix)

## Strategies that need this treatment

| Strategy | Config has long_only? | Live can short? | Backtest can short? |
|---|---|---|---|
